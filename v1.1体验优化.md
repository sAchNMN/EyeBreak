# v1.1 体验优化规划

目标：在不扩展产品范围的前提下，打磨 v1 已有功能的日常使用体验。

不做账号系统、云同步、每日报告、AI 分析、摄像头检测、完整番茄钟流程和 Windows 安装包。

## 优化总览

```text
┌──────┬──────────────┬────────────────────────────┬──────────────┐
│ 顺序 │ 模块         │ 优化点                     │ 交付结果     │
├──────┼──────────────┼────────────────────────────┼──────────────┤
│ 1    │ 悬浮倒计时   │ 贴边、拖动、隐藏反馈更稳   │ 减少误操作   │
│ 2    │ 提醒弹窗     │ 按钮语义、暂停调节更直观   │ 操作更清楚   │
│ 3    │ 设置窗口     │ 输入控件更像设置面板       │ 少输错       │
│ 4    │ 托盘菜单     │ 文案和状态更明确           │ 状态更可见   │
│ 5    │ 状态反馈     │ 暂停、离开、全屏显示统一   │ 用户不困惑   │
└──────┴──────────────┴────────────────────────────┴──────────────┘
```

## 推荐第一批

### 1. 悬浮倒计时

范围：

* 优化贴边后的窄条可触达性。
* 优化拖动到边缘时的吸附反馈。
* 保持现有贴边自动隐藏行为。
* 暂不处理多显示器，避免扩大范围。

验收点：

* 窄条更容易被鼠标唤出。
* 拖动靠近边缘时，吸附结果符合预期。
* 拖离边缘后，悬浮窗保持完整显示。
* 不影响暂停、离开、全屏状态显示。

涉及文件：

```text
app/floating_countdown.py
tests/test_floating_countdown.py
```

### 2. 提醒弹窗

范围：

* 让暂停时长调节更直观。
* 保留鼠标滚轮调节能力。
* 可以增加可见的 `- / +` 调节按钮。
* 退出按钮保持现有行为，是否加确认需单独决定。

验收点：

* 用户不用猜也能看出当前暂停时长。
* 暂停时长仍限制在 1 到 120 分钟。
* 点击暂停后，悬浮窗显示暂停倒计时。
* 跳过、倒计时结束、退出流程不回归。

涉及文件：

```text
app/reminder_window.py
tests/test_timer.py
```

### 3. 设置窗口

范围：

* 优化数字输入体验。
* 保留现有配置项，不新增产品能力。
* 保存成功后给出明确反馈。
* 非法输入时仍不覆盖 `config.json`。

验收点：

* 提醒间隔、休息时长、默认暂停和离开检测更容易填写。
* 保存合法值后配置写入成功。
* 非法值提示清楚。
* 保存提醒间隔后，下次提醒倒计时立即使用新值。

涉及文件：

```text
app/settings_window.py
tests/test_settings_window.py
```

## 第二批

### 4. 托盘菜单

范围：

* 优化“开关悬浮窗”文案。
* 让菜单文案更贴近当前状态。
* 保持现有 pystray 实现，不新增依赖。

验收点：

* 勾选状态和悬浮窗显示状态一致。
* 暂停、恢复、立即休息、设置和退出不回归。

涉及文件：

```text
app/tray.py
tests/test_tray.py
```

### 5. 状态反馈统一

范围：

* 统一 `下次护眼提醒`、`暂停中`、`已离开`、`全屏中` 的显示规则。
* 统一不同状态下的颜色和时间文本。
* 保持状态优先级：暂停优先于离开和全屏。

验收点：

* 暂停时显示暂停剩余时间。
* 离开时显示 `已离开` 和 `--:--`。
* 全屏时显示 `全屏中` 和 `--:--`。
* 返回正常状态后重新开始下一轮提醒计时。

涉及文件：

```text
app/timer.py
app/floating_countdown.py
tests/test_timer.py
tests/test_idle.py
tests/test_fullscreen.py
```

## 执行顺序

                 │            ├── 提醒弹窗

                 └── 第二批 ──┬── 托盘菜单
