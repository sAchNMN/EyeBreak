## 1. 执行摘要

EyeBreak 当前是一个功能完整的 Windows 桌面护眼提醒工具，采用 Python + Tkinter + pystray 技术栈。项目代码量约 1500 行，已发布 v1 版本并通过验收。

但现有架构存在一个核心问题：**`ReminderTimer` 是一个 God Class**，承担了计时调度、空闲检测、全屏检测、提醒展示、设置管理、托盘管理、状态持久化、暂停恢复等全部职责，直接依赖 10 个模块，导致高耦合、低内聚、难以扩展和测试。

**重构目标**：将单点 God Class 拆解为四层架构 + 事件总线 + 正式状态机，实现：

- **扩展性**：新增功能（如每日报告、云同步）只需订阅事件或添加新模块，不修改现有逻辑
- **可测试性**：核心业务逻辑零 UI/平台依赖，可独立单元测试
- **可靠性**：正式状态机消除布尔标志组合爆炸，状态转换可穷举验证
- **可维护性**：每层职责单一，修改影响范围可控

**关键约束**：不引入新外部依赖，仅使用 Python 标准库（`abc`、`dataclasses`、`enum`、`typing`）。

## 2. 现有架构分析

### 2.1 架构现状

```
main.py
  └── ReminderTimer (God Class, 310 行)
        ├── 直接实例化 → FloatingCountdownWindow (UI)
        ├── 直接实例化 → ReminderWindow (UI)
        ├── 直接实例化 → SettingsWindow (UI)
        ├── 直接实例化 → TrayIcon (UI)
        ├── 直接调用   → get_idle_seconds() (平台)
        ├── 直接调用   → is_foreground_window_fullscreen() (平台)
        ├── 直接调用   → set_autostart() / is_autostart_enabled() (平台)
        ├── 直接调用   → save_config() / load_config() (持久化)
        ├── 直接调用   → save_app_state() / load_app_state() (持久化)
        └── 直接调用   → apply_window_icon() / ensure_icon_file() (平台)
```

### 2.2 识别出的核心问题

| # | 问题 | 严重性 | 影响 |
|---|------|--------|------|
| P1 | **God Class 反模式** | 严重 | `ReminderTimer` 310 行承担 10+ 职责，任何修改都可能引发连锁反应 |
| P2 | **布尔标志管理状态** | 严重 | `is_showing_reminder` + `_was_idle` + `_was_fullscreen` + `paused_until` 组合出 16+ 种隐式状态，无法穷举测试 |
| P3 | **无事件解耦** | 高 | 组件通过直接回调通信，新增订阅者需修改发布方代码 |
| P4 | **无平台抽象层** | 高 | `idle.py`/`fullscreen.py`/`autostart.py` 直接被业务逻辑调用，无法 Mock 测试 |
| P5 | **UI 与业务逻辑混合** | 高 | `_tick()` 方法同时处理计时逻辑、空闲检测、全屏检测、UI 更新，职责交织 |
| P6 | **无依赖注入** | 中 | 所有依赖在类内部直接实例化，无法替换实现进行测试 |
| P7 | **错误处理不一致** | 中 | 部分操作静默失败（`save_app_state` 捕获 OSError 后 return），部分直接抛出 |

### 3.2 依赖规则

```
UI 层 ──依赖──→ 核心域层
核心域层 ←──依赖倒置──→ 平台抽象层 (通过 Protocol)
核心域层 ←──依赖倒置──→ 基础设施层 (通过 Protocol)
UI 层 ──不直接依赖──→ 平台/基础设施层
```

**核心原则**：依赖方向永远向下，下层不知道上层的存在。核心域层通过 Protocol（抽象接口）声明它需要什么能力，由 DI 容器在启动时注入具体实现。

## 4. 核心模式详解

### 4.1 事件总线（Event Bus）

**目标**：用发布/订阅模式替代直接回调，实现组件间松耦合。

```python
# app/core/events.py
from __future__ import annotations
from dataclasses import dataclass
from collections.abc import Callable
from enum import Enum, auto


class EventType(Enum):
    """所有事件类型的枚举，新增事件只需在此添加。"""
    COUNTDOWN_TICK = auto()          # 每秒倒计时刷新
    REMINDER_DUE = auto()            # 提醒时间到
    REMINDER_DISMISSED = auto()      # 提醒被关闭（跳过/超时）
    PAUSE_REQUESTED = auto()         # 用户请求暂停
    PAUSE_STARTED = auto()           # 暂停已生效
    PAUSE_EXPIRED = auto()           # 暂停到期，自动恢复
    RESUME_REQUESTED = auto()        # 用户请求恢复
    IDLE_DETECTED = auto()           # 检测到用户离开
    IDLE_RETURNED = auto()           # 用户返回
    FULLSCREEN_DETECTED = auto()     # 检测到全屏应用
    FULLSCREEN_EXITED = auto()       # 全屏应用退出
    CONFIG_CHANGED = auto()          # 配置已变更
    FLOATING_TOGGLED = auto()        # 悬浮窗开关切换
    AUTOSTART_TOGGLED = auto()       # 开机自启开关切换
    SETTINGS_OPENED = auto()         # 设置窗口打开请求
    EXIT_REQUESTED = auto()          # 退出请求


@dataclass(frozen=True)
class Event:
    """基础事件类，具体事件可继承添加字段。"""
    type: EventType
    payload: dict | None = None


class EventBus:
    """轻量级事件总线，支持发布/订阅。

    设计决策：
    - 同步分发：桌面应用单线程，无需异步队列
    - 订阅者异常不阻断其他订阅者：一个 UI 组件出错不影响其他
    - 不引入外部依赖：纯标准库实现
    """

    def __init__(self) -> None:
        self._subscribers: dict[EventType, list[Callable[[Event], None]]] = {}

    def subscribe(self, event_type: EventType, handler: Callable[[Event], None]) -> None:
        self._subscribers.setdefault(event_type, []).append(handler)

    def publish(self, event: Event) -> None:
        for handler in self._subscribers.get(event.type, []):
            try:
                handler(event)
            except Exception:
                # 记录但不中断其他订阅者
                # 生产环境可接入 logging
                pass
```

**使用示例**：

```python
# UI 层订阅事件，不再需要 ReminderTimer 直接调用
class FloatingCountdownView:
    def __init__(self, event_bus: EventBus) -> None:
        event_bus.subscribe(EventType.COUNTDOWN_TICK, self._on_tick)
        event_bus.subscribe(EventType.PAUSE_STARTED, self._on_pause)
        event_bus.subscribe(EventType.IDLE_DETECTED, self._on_idle)

    def _on_tick(self, event: Event) -> None:
        info = event.payload["countdown_info"]
        self._label.configure(text=info.formatted_remaining)

    def _on_pause(self, event: Event) -> None:
        self._status_label.configure(text="暂停中", fg="#fbbf24")

# 核心域层发布事件，不知道谁在监听
class TimerEngine:
    def __init__(self, event_bus: EventBus) -> None:
        self._event_bus = event_bus

    def _tick(self) -> None:
        remaining = self._next_reminder_at - time.monotonic()
        if remaining <= 0:
            self._event_bus.publish(Event(EventType.REMINDER_DUE))
        else:
            self._event_bus.publish(Event(
                EventType.COUNTDOWN_TICK,
                {"countdown_info": CountdownInfo(remaining=remaining)}
            ))
```

### 4.2 状态机（State Machine）

**目标**：用正式状态机替换布尔标志组合，使状态转换显式、可穷举、可测试。

```python
# app/core/state_machine.py
from __future__ import annotations
from enum import Enum, auto
from dataclasses import dataclass


class AppState(Enum):
    """应用状态的唯一真源，任何时刻只处于一个状态。"""
    RUNNING = auto()           # 正常倒计时
    IDLE = auto()              # 用户离开
    FULLSCREEN = auto()        # 全屏应用运行中
    PAUSED = auto()            # 手动暂停
    SHOWING_REMINDER = auto()  # 提醒弹窗打开
    EXITED = auto()            # 程序退出


@dataclass
class StateTransition:
    """状态转换规则：从某状态，经某触发条件，到另一状态。"""
    from_state: AppState
    trigger: str               # 触发条件名称
    to_state: AppState


# 状态转换表：所有合法转换的穷举定义
TRANSITIONS: list[StateTransition] = [
    # Running 的转换
    StateTransition(AppState.RUNNING, "idle_detected", AppState.IDLE),
    StateTransition(AppState.RUNNING, "fullscreen_detected", AppState.FULLSCREEN),
    StateTransition(AppState.RUNNING, "pause_requested", AppState.PAUSED),
    StateTransition(AppState.RUNNING, "reminder_due", AppState.SHOWING_REMINDER),
    StateTransition(AppState.RUNNING, "exit_requested", AppState.EXITED),

    # Idle 的转换
    StateTransition(AppState.IDLE, "idle_returned", AppState.RUNNING),
    StateTransition(AppState.IDLE, "exit_requested", AppState.EXITED),

    # Fullscreen 的转换
    StateTransition(AppState.FULLSCREEN, "fullscreen_exited", AppState.RUNNING),
    StateTransition(AppState.FULLSCREEN, "exit_requested", AppState.EXITED),

    # Paused 的转换
    StateTransition(AppState.PAUSED, "pause_expired", AppState.RUNNING),
    StateTransition(AppState.PAUSED, "resume_requested", AppState.RUNNING),
    StateTransition(AppState.PAUSED, "exit_requested", AppState.EXITED),

    # ShowingReminder 的转换
    StateTransition(AppState.SHOWING_REMINDER, "reminder_dismissed", AppState.RUNNING),
    StateTransition(AppState.SHOWING_REMINDER, "pause_from_reminder", AppState.PAUSED),
    StateTransition(AppState.SHOWING_REMINDER, "exit_requested", AppState.EXITED),
]


class StateMachine:
    """正式状态机：只允许预定义的合法转换。

    核心价值：
    - 消除布尔标志组合爆炸（4 个标志 → 16+ 隐式状态 → 6 个显式状态）
    - 非法转换立即报错，而不是产生不可预测的行为
    - 状态转换表可穷举测试
    """

    def __init__(self, initial_state: AppState = AppState.RUNNING) -> None:
        self._state = initial_state
        self._transition_map: dict[tuple[AppState, str], AppState] = {
            (t.from_state, t.trigger): t.to_state
            for t in TRANSITIONS
        }

    @property
    def current(self) -> AppState:
        return self._state

    def transition(self, trigger: str) -> AppState:
        """尝试状态转换。非法转换抛出 ValueError。"""
        key = (self._state, trigger)
        if key not in self._transition_map:
            raise ValueError(
                f"非法状态转换：{self._state.name} + {trigger} "
                f"(合法触发器: {[t for s, t in self._transition_map if s == self._state]})"
            )
        self._state = self._transition_map[key]
        return self._state

    def can_transition(self, trigger: str) -> bool:
        return (self._state, trigger) in self._transition_map
```

**对比**：

| 维度 | 现有布尔标志 | 正式状态机 |
|------|-------------|-----------|
| 状态数量 | 4 个标志 → 16 种隐式组合 | 6 个显式状态 |
| 非法状态 | 不报错，行为不可预测 | 立即抛 ValueError |
| 可测试性 | 无法穷举组合 | 转换表可 100% 覆盖 |
| 新增状态 | 修改多个布尔标志 | 添加枚举值 + 转换规则 |

### 4.3 依赖注入容器（DI Container）

**目标**：在单一入口组装所有依赖，组件不自行实例化依赖，支持测试替换。

```python
# app/container.py
from __future__ import annotations
from app.core.events import EventBus
from app.core.state_machine import StateMachine
from app.core.timer_engine import TimerEngine
from app.core.models import AppConfig, AppState as AppRuntimeState
from app.platform.idle_detector import WindowsIdleDetector
from app.platform.fullscreen_detector import WindowsFullscreenDetector
from app.platform.autostart_manager import WindowsAutostartManager
from app.infra.config_repository import ConfigRepository
from app.infra.state_repository import StateRepository
from app.infra.paths import PathResolver


class Container:
    """依赖注入容器：应用启动时一次性组装所有依赖。

    设计决策：
    - 不引入第三方 DI 框架（如 dependency-injector）
    - 手动组装，显式可读，适合项目规模
    - 每个组件只声明它需要什么（构造函数参数），不关心实现从哪来
    """

    def __init__(self) -> None:
        # 1. 基础设施层（无依赖）
        self.path_resolver = PathResolver()
        self.config_repo = ConfigRepository(self.path_resolver)
        self.state_repo = StateRepository(self.path_resolver)

        # 2. 核心域层（依赖基础设施通过 Protocol）
        self.event_bus = EventBus()
        self.state_machine = StateMachine()
        self.config = self.config_repo.load()
        self.runtime_state = self.state_repo.load()

        self.timer_engine = TimerEngine(
            event_bus=self.event_bus,
            state_machine=self.state_machine,
            config=self.config,
        )

        # 3. 平台抽象层（被核心域层通过 Protocol 消费）
        self.idle_detector = WindowsIdleDetector()
        self.fullscreen_detector = WindowsFullscreenDetector()
        self.autostart_manager = WindowsAutostartManager()

        # 4. UI 层（最后创建，依赖核心域层事件总线）
        # UI 组件在 app.py 中创建，因为需要 Tk root

    def reload_config(self) -> None:
        """配置变更后重新加载。"""
        self.config = self.config_repo.load()
        self.timer_engine.update_config(self.config)
```

### 4.4 平台抽象层（Protocol 接口）

**目标**：核心域层通过 Protocol 接口声明能力需求，平台层提供具体实现，测试时可注入 Mock。

```python
# app/platform/protocols.py
from __future__ import annotations
from typing import Protocol


class IdleDetectorProtocol(Protocol):
    """空闲检测能力接口。"""

    def get_idle_seconds(self) -> float:
        """返回用户最后一次输入到现在的秒数。"""
        ...


class FullscreenDetectorProtocol(Protocol):
    """全屏检测能力接口。"""

    def is_foreground_fullscreen(self) -> bool:
        """返回当前前台窗口是否全屏。"""
        ...


class AutostartManagerProtocol(Protocol):
    """开机自启管理能力接口。"""

    def is_enabled(self) -> bool:
        """返回自启是否已启用。"""
        ...

    def set_enabled(self, enabled: bool) -> None:
        """启用或禁用自启。"""
        ...


# app/platform/idle_detector.py
from __future__ import annotations
import sys


class WindowsIdleDetector:
    """Windows 平台空闲检测实现。

    核心域层只依赖 IdleDetectorProtocol，不知道也不关心
    底层是用 ctypes、GetLastInputInfo 还是其他方式实现。
    """

    def get_idle_seconds(self) -> float:
        if sys.platform != "win32":
            return 0.0
        try:
            import ctypes
            from ctypes import wintypes

            class LASTINPUTINFO(ctypes.Structure):
                _fields_ = [
                    ("cbSize", wintypes.UINT),
                    ("dwTime", wintypes.DWORD),
                ]

            lii = LASTINPUTINFO()
            lii.cbSize = ctypes.sizeof(LASTINPUTINFO)
            if not ctypes.windll.user32.GetLastInputInfo(ctypes.byref(lii)):
                return 0.0
            return (ctypes.windll.kernel32.GetTickCount() - lii.dwTime) / 1000.0
        except (AttributeError, OSError, TypeError):
            return 0.0


# 测试时可以轻松替换
class MockIdleDetector:
    """测试用 Mock：可预设返回值。"""

    def __init__(self, idle_seconds: float = 0.0) -> None:
        self._idle_seconds = idle_seconds

    def get_idle_seconds(self) -> float:
        return self._idle_seconds

    def set_idle_seconds(self, seconds: float) -> None:
        self._idle_seconds = seconds
```

### 5.1 TimerEngine（提醒调度引擎）

从 `ReminderTimer` 中提取纯业务逻辑，剥离所有 UI 和平台代码。

```python
# app/core/timer_engine.py
from __future__ import annotations
import time
from app.core.events import EventBus, Event, EventType
from app.core.state_machine import StateMachine, AppState
from app.core.models import AppConfig, CountdownInfo
from app.platform.protocols import IdleDetectorProtocol, FullscreenDetectorProtocol


class TimerEngine:
    """提醒调度引擎：核心域层组件。

    职责：
    - 管理提醒倒计时
    - 检测空闲/全屏状态，驱动状态机转换
    - 每秒发布 COUNTDOWN_TICK 事件
    - 到时间发布 REMINDER_DUE 事件

    不职责：
    - 不创建/管理 UI 窗口（由 UI 层订阅事件处理）
    - 不直接调用平台 API（通过 Protocol 接口）
    - 不持久化数据（由基础设施层处理）
    """

    def __init__(
        self,
        event_bus: EventBus,
        state_machine: StateMachine,
        config: AppConfig,
        idle_detector: IdleDetectorProtocol | None = None,
        fullscreen_detector: FullscreenDetectorProtocol | None = None,
    ) -> None:
        self._event_bus = event_bus
        self._sm = state_machine
        self._config = config
        self._idle_detector = idle_detector
        self._fullscreen_detector = fullscreen_detector
        self._next_reminder_at = 0.0
        self._paused_until = 0.0

    def start(self) -> None:
        """启动引擎，安排第一次提醒。"""
        self._schedule_next_reminder()

    def _schedule_next_reminder(self) -> None:
        self._next_reminder_at = (
            time.monotonic() + self._config.reminder_interval_minutes * 60
        )

    def tick(self) -> None:
        """每秒调用一次，驱动所有状态检测和事件发布。

        这是从 ReminderTimer._tick() 提取的纯逻辑版本，
        不包含任何 Tkinter 调用。
        """
        if self._sm.current == AppState.EXITED:
            return

        now = time.monotonic()

        # 暂停状态：只发倒计时事件
        if self._sm.current == AppState.PAUSED:
            if self._paused_until <= now:
                self._sm.transition("pause_expired")
                self._event_bus.publish(Event(EventType.PAUSE_EXPIRED))
                self._schedule_next_reminder()
            else:
                self._publish_countdown(
                    self._paused_until - now, status="paused"
                )
            return

        # 空闲检测
        if self._idle_detector and self._should_check_idle():
            if self._idle_detector.get_idle_seconds() >= self._config.idle_threshold_minutes * 60:
                if self._sm.current == AppState.RUNNING:
                    self._sm.transition("idle_detected")
                    self._event_bus.publish(Event(EventType.IDLE_DETECTED))
                return
            elif self._sm.current == AppState.IDLE:
                self._sm.transition("idle_returned")
                self._event_bus.publish(Event(EventType.IDLE_RETURNED))
                self._schedule_next_reminder()

        # 全屏检测
        if self._fullscreen_detector and self._config.fullscreen_detection_enabled:
            if self._fullscreen_detector.is_foreground_fullscreen():
                if self._sm.current == AppState.RUNNING:
                    self._sm.transition("fullscreen_detected")
                    self._event_bus.publish(Event(EventType.FULLSCREEN_DETECTED))
                return
            elif self._sm.current == AppState.FULLSCREEN:
                self._sm.transition("fullscreen_exited")
                self._event_bus.publish(Event(EventType.FULLSCREEN_EXITED))
                self._schedule_next_reminder()

        # 正常倒计时
        remaining = self._next_reminder_at - now
        if remaining <= 0:
            self._sm.transition("reminder_due")
            self._event_bus.publish(Event(EventType.REMINDER_DUE))
        else:
            self._publish_countdown(remaining, status="running")

    def pause(self, minutes: int) -> None:
        """用户请求暂停。"""
        self._paused_until = time.monotonic() + minutes * 60
        self._next_reminder_at = (
            self._paused_until + self._config.reminder_interval_minutes * 60
        )
        if self._sm.current in (AppState.RUNNING, AppState.SHOWING_REMINDER):
            trigger = "pause_requested" if self._sm.current == AppState.RUNNING else "pause_from_reminder"
            self._sm.transition(trigger)
            self._event_bus.publish(Event(
                EventType.PAUSE_STARTED,
                {"paused_until": self._paused_until}
            ))

    def resume(self) -> None:
        """用户请求恢复。"""
        if self._sm.current == AppState.PAUSED:
            self._sm.transition("resume_requested")
            self._paused_until = 0.0
            self._schedule_next_reminder()
            self._event_bus.publish(Event(EventType.RESUME_REQUESTED))

    def dismiss_reminder(self) -> None:
        """提醒被关闭。"""
        if self._sm.current == AppState.SHOWING_REMINDER:
            self._sm.transition("reminder_dismissed")
            self._event_bus.publish(Event(EventType.REMINDER_DISMISSED))
            self._schedule_next_reminder()

    def request_exit(self) -> None:
        """请求退出。"""
        self._sm.transition("exit_requested")
        self._event_bus.publish(Event(EventType.EXIT_REQUESTED))

    def update_config(self, config: AppConfig) -> None:
        """配置变更后更新。"""
        self._config = config
        if self._sm.current == AppState.PAUSED:
            self._next_reminder_at = (
                self._paused_until + config.reminder_interval_minutes * 60
            )
        elif self._sm.current == AppState.RUNNING:
            self._schedule_next_reminder()
        self._event_bus.publish(Event(EventType.CONFIG_CHANGED))

    def _should_check_idle(self) -> bool:
        return (
            self._config.idle_threshold_minutes > 0
            and self._sm.current in (AppState.RUNNING, AppState.IDLE)
        )

    def _publish_countdown(self, remaining: float, status: str) -> None:
        self._event_bus.publish(Event(
            EventType.COUNTDOWN_TICK,
            {"countdown_info": CountdownInfo(remaining=remaining, status=status)}
        ))
```

```python
    """应用配置（不可变，变更时创建新实例）。"""

    enabled: bool = True
    edge: str | None = "right"
    x: int = 0
    y: int | None = None


@dataclass
class CountdownInfo:
    """倒计时信息，随 COUNTDOWN_TICK 事件传递。"""
    remaining: float
    status: str  # "running" | "paused" | "idle" | "fullscreen"

    @property
    def formatted_remaining(self) -> str:
        clamped = max(0, int(self.remaining + 0.999))
        minutes, seconds = divmod(clamped, 60)
        return f"{minutes:02d}:{seconds:02d}"
```

```python
# app/infra/config_repository.py
from __future__ import annotations
import json
from pathlib import Path
from typing import Any
from app.core.models import AppConfig
from app.infra.paths import PathResolver


class ConfigRepository:
    """配置持久化仓库。

    核心域层通过 Protocol 声明需要"加载/保存配置"的能力，
    此类提供 JSON 文件实现，未来可替换为 SQLite 或云端存储。
    """

    def __init__(self, path_resolver: PathResolver) -> None:
        self._path = path_resolver.config_path()

    def load(self) -> AppConfig:
        if not self._path.exists():
            self.save(AppConfig())
            return AppConfig()

        try:
            raw = json.loads(self._path.read_text(encoding="utf-8"))
        except (OSError, json.JSONDecodeError):
            return AppConfig()

        if not isinstance(raw, dict):
            return AppConfig()

        return AppConfig(
            reminder_interval_minutes=self._positive(
                raw.get("reminder_interval_minutes"),
                AppConfig().reminder_interval_minutes,
            ),
            break_duration_seconds=int(self._positive(
                raw.get("break_duration_seconds"),
                AppConfig().break_duration_seconds,
            )),
            pause_minutes=self._positive(
                raw.get("pause_minutes"),
                AppConfig().pause_minutes,
            ),
            idle_threshold_minutes=self._non_negative(
                raw.get("idle_threshold_minutes"),
                AppConfig().idle_threshold_minutes,
            ),
            fullscreen_detection_enabled=self._bool(
                raw.get("fullscreen_detection_enabled"),
                AppConfig().fullscreen_detection_enabled,
            ),
        )

    def save(self, config: AppConfig) -> None:
        payload = {
            "reminder_interval_minutes": config.reminder_interval_minutes,
            "break_duration_seconds": config.break_duration_seconds,
            "pause_minutes": config.pause_minutes,
            "idle_threshold_minutes": config.idle_threshold_minutes,
            "fullscreen_detection_enabled": config.fullscreen_detection_enabled,
        }
        self._path.write_text(
            json.dumps(payload, indent=2), encoding="utf-8"
        )

    @staticmethod
    def _positive(value: Any, fallback: float) -> float:
        if isinstance(value, bool):
            return fallback
        try:
            parsed = float(value)
        except (TypeError, ValueError):
            return fallback
        return parsed if parsed > 0 else fallback

    @staticmethod
    def _non_negative(value: Any, fallback: float) -> float:
        if isinstance(value, bool):
            return fallback
        try:
            parsed = float(value)
        except (TypeError, ValueError):
            return fallback
        return parsed if parsed >= 0 else fallback

    @staticmethod
    def _bool(value: Any, fallback: bool) -> bool:
        return value if isinstance(value, bool) else fallback
```

## 6. 迁移路径

### 6.1 分阶段迁移策略

重构不追求一步到位，按以下 4 个阶段逐步推进，每个阶段都可独立交付和验收。

```
阶段 1 (基础设施)  →  阶段 2 (核心域)  →  阶段 3 (平台抽象)  →  阶段 4 (UI 重构)
    1-2 天               2-3 天               1 天                  2-3 天
```

### 阶段 1：建立目录结构与领域模型

**目标**：创建新的目录结构，提取领域模型，不改变现有运行行为。

**步骤**：
1. 创建 `app/core/`、`app/platform/`、`app/ui/`、`app/infra/` 目录
2. 将 `app/config.py` 中的 `AppConfig` 移到 `app/core/models.py`
3. 将 `app/state.py` 中的 `AppState` 拆分为 `FloatingPosition`（领域模型）和运行时状态
4. 将 `app/paths.py` 移到 `app/infra/paths.py`
5. 在 `app/core/events.py` 中实现 `EventBus` 和 `EventType`
6. 在 `app/core/state_machine.py` 中实现 `StateMachine`
7. 现有代码通过 re-export 保持兼容：`app/config.py` → `from app.core.models import AppConfig`

**验收标准**：
- `python main.py` 正常运行
- `python -m pytest -q tests` 全部通过
- 新代码有独立单元测试

### 阶段 2：提取 TimerEngine

**目标**：从 `ReminderTimer._tick()` 中提取纯业务逻辑到 `TimerEngine`。

**步骤**：
1. 创建 `app/core/timer_engine.py`，实现 `TimerEngine` 类
2. `TimerEngine` 只依赖 `EventBus`、`StateMachine`、`AppConfig` 和 Protocol 接口
3. 为 `TimerEngine` 编写完整单元测试（使用 Mock 检测器）
4. `ReminderTimer` 改为委托 `TimerEngine` 处理逻辑，自己只负责 UI 线程调度
5. 逐步将 `ReminderTimer` 的方法委托给 `TimerEngine`：`_pause` → `timer_engine.pause()`，`_resume` → `timer_engine.resume()` 等

**验收标准**：
- `TimerEngine` 有独立测试，覆盖所有状态转换
- `python main.py` 正常运行
- `python -m pytest -q tests` 全部通过

### 阶段 3：平台抽象层

**目标**：为平台功能建立 Protocol 接口，现有实现适配接口。

**步骤**：
1. 在 `app/platform/protocols.py` 中定义 `IdleDetectorProtocol`、`FullscreenDetectorProtocol`、`AutostartManagerProtocol`
2. 将 `app/idle.py` 移到 `app/platform/idle_detector.py`，实现 `IdleDetectorProtocol`
3. 将 `app/fullscreen.py` 移到 `app/platform/fullscreen_detector.py`
4. 将 `app/autostart.py` 移到 `app/platform/autostart_manager.py`
5. `TimerEngine` 通过 Protocol 接口使用检测器，不再直接导入具体实现
6. 创建 `app/container.py`，在容器中组装依赖

**验收标准**：
- `TimerEngine` 测试中可注入 Mock 检测器
- `python main.py` 正常运行
- `python -m pytest -q tests` 全部通过

### 阶段 4：UI 层事件驱动重构

**目标**：UI 组件通过订阅事件更新视图，不再被 `ReminderTimer` 直接调用。

**步骤**：
1. 将 `app/floating_countdown.py` 移到 `app/ui/floating_countdown.py`
2. `FloatingCountdownWindow` 构造函数接收 `EventBus`，订阅 `COUNTDOWN_TICK`、`PAUSE_STARTED` 等事件
3. 将 `app/tray.py` 移到 `app/ui/tray_icon.py`，用户操作改为发布事件（`PAUSE_REQUESTED`、`EXIT_REQUESTED` 等）
4. 将 `app/reminder_window.py` 移到 `app/ui/reminder_window.py`
5. 将 `app/settings_window.py` 移到 `app/ui/settings_window.py`
6. `ReminderTimer` 简化为 `App` 类：创建容器、创建 Tk root、创建 UI 组件、启动 `TimerEngine`
7. 删除旧的 `app/timer.py`

**验收标准**：
- UI 组件只依赖 `EventBus` 和领域模型
- `python main.py` 正常运行，所有验收清单项通过

## 8. 风险评估

### 8.1 技术风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| Tkinter 线程安全：事件总线回调可能不在 UI 线程执行 | 中 | 高 | EventBus 的 publish 通过 `root.after(0, ...)` 调度到 UI 线程；或引入 `UIEventBus` 子类自动调度 |
| 状态机与现有测试不兼容 | 低 | 中 | 阶段 1-2 保持 re-export 兼容层，现有测试不改 |
| 迁移期间引入回归 bug | 中 | 高 | 每阶段完成后运行完整测试 + 手动验收，不跳过 |
| 过度设计：项目规模可能不需要完整四层架构 | 低 | 中 | 遵循 AGENTS.md 依赖策略：标准库优先，不过度抽象 |

### 8.2 项目约束

| 约束 | 说明 |
|------|------|
| **不修改源代码** | 本文档是设计方案，实际代码修改需用户确认后执行 |
| **不引入新依赖** | 仅使用 Python 标准库（`abc`, `dataclasses`, `enum`, `typing`） |
| **AGENTS.md 规则** | 每阶段完成后更新 `HANDOFF.md`，测试通过后才能提交 |
| **验收流程** | 每阶段需用户明确说"验收没有问题"才能进入下一阶段 |

### 8.3 关键设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 事件分发模型 | 同步分发 | 桌面应用单线程，无需异步队列；简单可靠 |
| 状态机实现 | 手写转换表 | 项目规模小，不需要 `transitions` 等第三方库 |
| DI 容器 | 手动组装 | 显式可读，不引入 `dependency-injector`；项目规模适合 |
| Protocol vs ABC | `typing.Protocol` | 鸭子类型，现有类无需继承基类即可适配 |
| frozen dataclass | `AppConfig` 使用 `frozen=True` | 配置不可变，变更时创建新实例，避免意外修改 |

## 附录 B：事件流示例

以"用户点击暂停"为例，对比现有架构和目标架构的执行流：

**现有架构**：
```
TrayIcon 暂停按钮
  → ReminderTimer._pause(minutes)        # 直接调用
    → self.state.paused_until = ...      # 修改状态
    → self._update_countdown_display()   # 直接更新 UI
```

**目标架构**：
```
TrayIconView 暂停按钮
  → EventBus.publish(PAUSE_REQUESTED)    # 发布事件
    → TimerEngine.on_pause_requested()   # 核心层处理
      → StateMachine.transition("pause_requested")
      → EventBus.publish(PAUSE_STARTED) # 发布结果事件
        → FloatingCountdownView 更新显示  # UI 订阅者自动响应
        → TrayIconView 更新菜单状态       # 另一个订阅者
        → (未来) ReportCollector 记录暂停  # 新增订阅者，零修改
```
